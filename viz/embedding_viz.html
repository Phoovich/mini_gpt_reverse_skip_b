The embedding matrix has one learned row per vocabulary token. We take the row for our token ID. These 8 numbers represent what this token means — learned during training.

Step 3 — Sinusoidal Positional Encoding  position-based

A fixed, mathematical pattern is added to tell the model where this token sits in the sequence. It's not learned — same position always gives same values.

PE table (5 positions × 8 dims)
PE vector for selected position pos 0
Formula
PE(pos, 2i)   = sin(pos / 100002i/d)
PE(pos, 2i+1) = cos(pos / 100002i/d)
Example — selected position
Even dimensions (0,2,4,6) use sin — they start at 0.
Odd dimensions (1,3,5,7) use cos — they start at 1.
Higher dimensions change more slowly across positions.

Step 5 — Final Vector

This 8-dimensional vector is what enters the first Transformer layer. It uniquely represents both the token's identity and its position in the sequence.

Token:   Position:
Blue portion (embedding)
Encodes the semantic meaning of this token. The model learns during training what "h", "e", "l", "o" should mean.
Orange portion (PE)
Encodes the position. Always fixed by formula — never trained. Without this, the model couldn't tell position 2 from position 3.
Key Insight

Same token, different position → different final vector

"hello" contains 'l' at both position 2 and position 3. Their embeddings are identical — but the PE differs, so the combined vectors are different. The Transformer can distinguish them.

Summary

The three-part formula

🔤
Token Embedding
"What is this token?"

Learned row of 8 numbers unique to each vocabulary item. Same token always → same embedding.
Emb["l"] = const
📍
Positional Encoding
"Where is this token?"

Fixed sin/cos formula. Never learned. Same position always → same PE values.
PE(pos=2) = const
✨
Combined Vector
"What AND where?"

Embedding + PE gives every (token, position) pair a unique fingerprint that feeds into the Transformer.
Emb + PE = unique
Deep Dive

How PE values change across positions

Different dimensions oscillate at different speeds. Low dimensions change fast (encode fine-grained position); high dimensions change slowly (encode coarse position). Together they make every position unique.
