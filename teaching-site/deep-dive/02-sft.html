1. Maximum Likelihood Principle

เป้าหมายของ SFT คือหาค่า parameter θ ที่ทำให้โมเดลให้ probability สูงสุดแก่ training data กล่าวอีกนัยหนึ่งคือ maximize likelihood ของข้อมูล:

Maximum Likelihood Objective
\[ \theta^* = \arg\max_\theta \sum_{n=1}^{N} \sum_{t=1}^{T_n} \log p_\theta(y_t^{(n)} \mid x_{<t}^{(n)}) \]

สังเกตว่าเราใช้ log probability แทน probability ตรงๆ เพราะ: (1) log ทำให้ผลคูณกลายเป็นผลบวก — คำนวณง่ายกว่ามาก (2) log เป็น monotone — argmax ไม่เปลี่ยน (3) numerical stability — probability หลายตัวคูณกันจะ underflow

ใน code เราทำ gradient descent บน negative log-likelihood ซึ่งก็คือ cross-entropy loss:

\[ \mathcal{L}(\theta) = -\frac{1}{T} \sum_{t=1}^{T} \log p_\theta(y_t \mid x_{<t}) \]

2. Cross-Entropy Loss — อนุมานจาก MLE

Cross-entropy ระหว่าง distribution จริง p และ distribution ของโมเดล q นิยามว่า:

Cross-Entropy
\[ H(p, q) = -\sum_x p(x) \log q(x) \]

สำหรับ training data เป้าหมาย p คือ one-hot distribution (รู้ token ที่ถูกต้องแน่นอน) ดังนั้น:

\[ H(p, q) = -\log q(y_{\text{true}}) \]

Cross-entropy ยังเชื่อมกับ KL divergence ผ่าน: \( H(p, q) = H(p) + \text{KL}(p \| q) \) เพราะ H(p) คงที่ การ minimize cross-entropy ก็คือการ minimize KL ซึ่งก็คือทำให้โมเดล "ใกล้เคียง" distribution ข้อมูลมากที่สุด

ตัวอย่าง — input "tesbt", step 0

Target token = "t" (ตัวแรกของ reversed output)

โมเดล SFT ให้ P("t") = 0.82

Cross-entropy loss = \(-\log(0.82) \approx 0.198\)

โมเดล RL ที่ step 1 ให้ P("s") = 0.45 (ไม่มี b)

CE = \(-\log(0.45) \approx 0.799\) — สูงกว่า เพราะยังไม่มั่นใจ

ยิ่ง P(correct) สูง → \(-\log p\) ต่ำ → loss ต่ำ → gradient update เล็ก (โมเดลมั่นใจถูก ไม่ต้องปรับมาก)

3. Teacher Forcing

ระหว่าง training ที่ทุก time step t โมเดลจะได้รับ ground truth token ที่ถูกต้อง เป็น input แทนที่จะใช้ output ของตัวเอง — นี่คือ teacher forcing

สำหรับ sequence [<BOS>, h, e, l, l, o, <SEP>, o, l, l, e, h, <EOS>]:
input x = tokens 0..11 | target y = tokens 1..12 (shift-by-1)

Teacher Forcing vs Autoregressive — เปรียบเทียบแบบ interactive

กด "ถัดไป" เพื่อดูทีละก้าว — ฝั่งซ้าย = teacher forcing, ฝั่งขวา = autoregressive

Autoregressive (inference)

ข้อดีของ teacher forcing: ทุก position ใน sequence สามารถ compute loss พร้อมกัน (parallel training) ไม่ต้องรอ token ก่อนหน้าเสร็จ — ต่างจาก RNN ที่ต้อง sequential

4. Exposure Bias

Teacher forcing สร้าง distribution mismatch ระหว่าง training กับ inference:

  • Training: P(input at step t = ground truth token) = 1
  • Inference: P(input at step t = model output) = 1

ถ้า model generate token ผิดในก้าวแรก ก้าวถัดๆ ไปทั้งหมดจะได้รับ context ที่ผิด ซึ่ง model ไม่เคยเห็นระหว่าง training — error compounding

ทำไม RL ช่วย? rollout_with_logprobs() ใช้ categorical sampling ทำให้ model ได้รับ model's own outputs เป็น input ทุกก้าวระหว่าง training — ลด distribution mismatch

5. Dynamic Padding + collate_fn

ใน batch หนึ่ง sequences มีความยาวต่างกัน เช่น 7, 11, 15 tokens PyTorch DataLoader ต้องการ tensor shape คงที่ จึงต้อง pad ทุก sequence ให้ยาวเท่ากับ sequence ที่ยาวที่สุดใน batch (dynamic padding)

ตัวอย่าง batch
Batch มี 3 sequences ยาว 5, 8, 12 tokens
หลัง collate_fn: x shape = (3, 12), y shape = (3, 12)
seq0: tokens 0..4 จริง + PAD ที่ positions 5..11 (7 positions)
seq1: tokens 0..7 จริง + PAD ที่ positions 8..11 (4 positions)
seq2: tokens 0..11 จริง (ไม่ต้อง pad)

PAD positions ใน target y ต้องไม่มีผลต่อ gradient — ใช้ ignore_index=PAD_ID ใน F.cross_entropy เพื่อทำให้ gradient = 0 ที่ PAD positions ทั้งหมด

Gradient proof F.cross_entropy ด้วย ignore_index=0 (PAD_ID): loss = (1/N_valid) * Σ_{t: y_t ≠ PAD} −log p(y_t) ∂loss/∂logits_t = 0 ถ้า y_t = PAD = softmax(logits_t) − one_hot(y_t) ถ้า y_t ≠ PAD → PAD positions ไม่ดึง embedding หรือ parameter ใดๆ

6. Gradient Flow Per Position

ข้อได้เปรียบสำคัญของ Transformer เทียบกับ RNN คือ gradient ไหลย้อนกลับมาพร้อมกันทุก position ใน sequence ในขณะที่ RNN ต้องผ่าน chain ยาว (vanishing gradient problem)

7. Code — collate_fn และ compute_loss

mini_gpt_reverse.py:38–49 — collate_fn

38def collate_fn(batch):
39    xs, ys = zip(*batch)
40    max_len = max(len(x) for x in xs)
41
42    x_padded = torch.full((len(xs), max_len), PAD_ID, dtype=torch.long)
43    y_padded = torch.full((len(ys), max_len), PAD_ID, dtype=torch.long)
44
45    for i, (x, y) in enumerate(zip(xs, ys)):
46        x_padded[i, : len(x)] = x
47        y_padded[i, : len(y)] = y
48
49    return x_padded, y_padded
ทำไม
torch.full(..., PAD_ID) สร้าง tensor เต็มด้วย PAD ก่อน แล้วค่อย fill ด้วยข้อมูลจริง — ทำให้ positions ที่ไม่ได้ fill ยังคงเป็น PAD_ID = 0 โดยอัตโนมัติ ไม่ต้อง loop fill PAD แยก; รองรับ batch_first=True ที่ MiniGPT ต้องการ

mini_gpt_reverse.py:55–63 — compute_loss

55def compute_loss(model, x, y, device):
56    pad_mask = x == PAD_ID
57    logits = model(x, pad_mask=pad_mask)
58    loss = F.cross_entropy(
59        logits.reshape(-1, VOCAB_SIZE),
60        y.reshape(-1),
61        ignore_index=PAD_ID,
62    )
63    return loss
ทำไม
logits.reshape(-1, VOCAB_SIZE) แปลง (B,T,V) → (B×T, V) เพื่อให้ F.cross_entropy รับได้ (ต้องการ 2D input); y.reshape(-1) → (B×T,); ignore_index=PAD_ID ทำให้ loss และ gradient = 0 ที่ทุก position ที่ target = PAD — ป้องกันโมเดลเรียนรู้อะไรจาก padding context

8. Stop & Check

ทดสอบความเข้าใจ

Q1: ถ้า model ให้ P(correct) = 0.5 loss = เท่าไร?

เฉลย
\( -\log(0.5) = \log(2) \approx 0.693 \)
เปรียบเทียบ: P=0.82 → loss=0.198; P=0.1 → loss=2.303

Q2: Batch มี 3 sequences ยาว 5, 8, 12 tokens หลัง collate_fn tensor มีขนาดเท่าไร?

เฉลย
(3, 12) ทั้ง x และ y
seq0: positions 5–11 เป็น PAD (7 positions)
seq1: positions 8–11 เป็น PAD (4 positions)
seq2: ไม่ต้อง pad (ยาวที่สุดอยู่แล้ว)

Q3: ทำไม Exposure bias จึงเป็นปัญหาสำหรับ task นี้โดยเฉพาะ?

เฉลย
Task = generate reversed sequence; ถ้า generate token ผิดในก้าวแรก token ที่เหลือทั้งหมดจะ off-by-position — error สะสมรุนแรงมาก

RL แก้เพราะ rollout_with_logprobs() ใช้ model output เป็น input ทุกก้าว — training distribution ตรงกับ inference distribution มากขึ้น
