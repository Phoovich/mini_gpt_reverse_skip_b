Policy Gradient, REINFORCE, GRPO

ก่อนเริ่ม — จาก overview section 5:

  • KL penalty (λ=0.2) ป้องกัน catastrophic forgetting

1. RL Objective: J(θ) = E[R]

เป้าหมายของ RL คือหา policy parameter θ ที่ maximize expected reward ทุก trajectory τ:

RL Objective
\[ J(\theta) = \mathbb{E}_{\tau \sim \pi_\theta}[R(\tau)] = \sum_\tau p_\theta(\tau) \cdot R(\tau) \]

ใน task ของเรา trajectory คือ sequence ที่ generate ออกมา เช่น ["m","o","<EOS>"] และ reward คำนวณจาก compute_reward(seq, output)

ตัวอย่าง — 4 rollouts ของ "bomb"
J ≈ (5.30 + (−1.00) + 0.17 + 5.30) / 4 = 2.44
เป้าหมาย: optimize θ ให้ J สูงขึ้น = rollouts ที่ถูกต้องเกิดบ่อยขึ้น

2. ∇J(θ): Log-Derivative Trick

เราต้องการ gradient ∇J(θ) เพื่อ update θ แต่ J ขึ้นกับ p_θ(τ) ที่เปลี่ยนตาม θ — จะ differentiate อย่างไร?

Log-Derivative Trick
\[ \nabla_\theta p_\theta(\tau) = p_\theta(\tau) \cdot \nabla_\theta \log p_\theta(\tau) \]
Proof (3 lines) ให้ f = p_θ(τ) ∇_θ f = f · ∇_θ log f เพราะ d(log f)/dθ = (1/f) · df/dθ → df/dθ = f · d(log f)/dθ ∎

นำไปใช้กับ ∇J:

\[ \nabla_\theta J(\theta) = \sum_\tau \nabla_\theta p_\theta(\tau) \cdot R(\tau) = \sum_\tau p_\theta(\tau) \cdot \nabla_\theta \log p_\theta(\tau) \cdot R(\tau) \] \[ = \mathbb{E}_\tau\!\left[ R(\tau) \cdot \nabla_\theta \log p_\theta(\tau) \right] \]
ตัวอย่าง
\( \log p_\theta(\text{"mo"}\,\text{<EOS>}\mid\text{"bomb"}) \)
= log π(m|ctx₁) + log π(o|ctx₂) + log π(<EOS>|ctx₃)
= sum ของ log_prob ทุก step ← นี่คือสิ่งที่ rollout_with_logprobs() track

3. Factoring by Step

trajectory p_θ(τ) เขียนเป็น product ของ per-step policy probability (Markov property):

\[ p_\theta(\tau) = \prod_{t=1}^{T} \pi_\theta(a_t \mid s_t) \] \[ \log p_\theta(\tau) = \sum_{t=1}^{T} \log \pi_\theta(a_t \mid s_t) \] \[ \nabla_\theta \log p_\theta(\tau) = \sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(a_t \mid s_t) \]

ดังนั้น policy gradient estimator กลายเป็น:

\[ \nabla_\theta J(\theta) = \mathbb{E}\!\left[ \sum_t \nabla_\theta \log \pi_\theta(a_t|s_t) \cdot R(\tau) \right] \]

ใน code: log_prob_sum = torch.stack(log_probs).sum() แล้ว policy_loss = -log_prob_sum × advantage

4. Variance Reduction — Baseline Subtraction

ปัญหา: estimator R(τ)·∇log p มี variance สูงมาก เพราะ R(τ) กระโดดขึ้นลงตาม rollout — gradient noisy → เรียนช้า

Baseline Subtraction Theorem
สำหรับ baseline b ที่ไม่ขึ้นกับ τ: \[ \mathbb{E}_\tau\!\left[ \nabla_\theta \log p_\theta(\tau) \cdot b \right] = 0 \] ดังนั้น: \( \nabla J(\theta) = \mathbb{E}[(R(\tau) - b) \cdot \nabla \log p_\theta(\tau)] \) และ variance ลดลง
Proof (3 lines) 𝔼_τ[∇_θ log p_θ(τ) · b] = b · Σ_τ ∇_θ p_θ(τ) = b · ∇_θ (Σ_τ p_θ(τ)) — gradient กับ sum สลับได้ = b · ∇_θ(1) = 0 — เพราะ Σ_τ p_θ(τ) = 1 เสมอ ∎

5. REINFORCE with EMA Baseline

วิธีมาตรฐาน: ใช้ exponential moving average (EMA) ของ reward ที่ผ่านมาเป็น baseline:

\[ b_{t+1} = \beta \cdot b_t + (1 - \beta) \cdot R(\tau_t) \] Advantage: \( A(\tau_t) = R(\tau_t) - b_t \)

ข้อเสีย:

  • ต้องการ warm-up period — ช่วงแรก b ≈ 0 ไม่ช่วยลด variance
  • b อาจ lag กับ non-stationary reward distribution ระหว่าง training
  • ต้องเก็บ state ข้ามหลาย steps

6. GRPO — Group Relative Policy Optimization

แทน EMA baseline ด้วย group mean ของ G rollouts บน prompt เดียวกัน:

GRPO Advantage
\[ \bar{r} = \frac{1}{G}\sum_{i=1}^{G} r_i \qquad \sigma_r = \sqrt{\frac{1}{G}\sum_{i=1}^{G}(r_i-\bar{r})^2} \] \[ A_i = \frac{r_i - \bar{r}}{\sigma_r + \varepsilon} \]
ตัวอย่าง "bomb", G=4
Rewards: 5.30, −1.00, 0.17, 5.30
r̄ = (5.30 − 1.00 + 0.17 + 5.30) / 4 = 2.44
σ = √[(2.86² + 3.44² + 2.27² + 2.86²) / 4] = √(8.35) ≈ 2.89
A₁ = (5.30 − 2.44) / 2.89 ≈ +0.99
A₂ = (−1.00 − 2.44) / 2.89 ≈ −1.19
A₃ = (0.17 − 2.44) / 2.89 ≈ −0.79
A₄ = (5.30 − 2.44) / 2.89 ≈ +0.99

Bias note: group mean correlated กับ rollout ตัวเอง → technically slightly biased แต่ bias = O(1/G) → ที่ G=4 ยอมรับได้ใน practice (จาก GRPO paper, DeepSeekMath 2024)

Policy Loss
\[ \mathcal{L}_{\text{policy}}(\theta) = -\frac{1}{B \cdot G} \sum_{j=1}^{B} \sum_{i=1}^{G} \log p_\theta(\tau_{j,i}) \cdot A_{j,i} \]

7. KL Penalty — Trust Region

ถ้า optimize J(θ) โดยไม่มี constraint policy อาจ drift จาก SFT reference มากจนลืมการ reverse ป้องกันด้วย KL penalty (Lagrangian relaxation):

Total Loss
\[ \mathcal{L}(\theta) = \mathcal{L}_{\text{policy}} + \lambda \cdot \text{KL}(\pi_\theta \| \pi_{\text{ref}}) + \mathcal{L}_{\text{entropy}} \] \[ \text{KL}(\pi_\theta \| \pi_{\text{ref}}) = \mathbb{E}_{x}\!\left[ \sum_t \pi_\theta(a_t|x) \log \frac{\pi_\theta(a_t|x)}{\pi_{\text{ref}}(a_t|x)} \right] \]

Heritage: TRPO ใช้ natural gradient + KL hard constraint; PPO ใช้ clipped ratio (soft constraint); GRPO ใช้ KL penalty (Lagrangian form) — simpler, differentiable directly

ค่าที่ใช้: λ = kl_coef = 0.2, entropy_coef = 0.005 (entropy bonus ส่งเสริม exploration)

178def compute_kl_penalty(model, ref_model, seq, device):
179    """KL divergence penalty จาก reference SFT model"""
180    tokens = ["<BOS>"] + seq + ["<SEP>"]
181    ids = encode(tokens)
182    x = torch.tensor(ids, dtype=torch.long, device=device).unsqueeze(0)
183
184    pad_mask = x == PAD_ID
185    with torch.no_grad():
186        ref_logits = ref_model(x, pad_mask=pad_mask)
187    logits = model(x, pad_mask=pad_mask)
188
189    ref_log_probs = torch.log_softmax(ref_logits, dim=-1)
190    log_probs = torch.log_softmax(logits, dim=-1)
191    probs = torch.softmax(logits, dim=-1)
192
193    kl = (probs * (log_probs - ref_log_probs)).sum(dim=-1).mean()
194    return kl

10. Stop & Check

ทดสอบความเข้าใจ

Q1: group rewards = [3, 3, 3, 3] advantage = เท่าไร?

เฉลย
σ_r = 0; A = (3−3)/(0+1e-8) ≈ 0 สำหรับทุก rollout
→ policy loss = 0 → no gradient update
ε = 1e-8 ป้องกัน NaN (division by zero)
Insight: ถ้า G rollouts ได้ reward เท่ากันหมด model ไม่รู้ว่าต้องปรับอะไร

Q2: ทำไม kl_coef = 0.0 จึงเป็นปัญหา?

เฉลย
ไม่มี KL constraint → model optimize reward โดยไม่มี "anchor"
อาจ converge ไปที่: generate <EOS> ทันที (reward = −0.2 ดีกว่า reverse ผิด)
หรือ policy degenerate อื่นๆ ที่ reward สูงแต่ไม่ใช่ intent
KL ป้องกัน catastrophic forgetting ของ reversal behavior

Q3: GRPO ด้วย G=1 (1 rollout per prompt) ทำงานไหม?

เฉลย
ไม่ทำงาน: baseline = r̄ = R(τ) เอง → A = (R−R)/std = 0/σ = 0
→ policy loss = 0 → no gradient
GRPO ต้องการ G ≥ 2 เพื่อ compute meaningful group variance
G = 4 ใน project นี้เป็น practical tradeoff ระหว่าง variance reduction และ compute cost
