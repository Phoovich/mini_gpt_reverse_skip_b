1. ปัญหาที่ Attention แก้

Embedding ปกติ (fixed lookup) ให้ vector เดิมสำหรับคำเดิมเสมอ ไม่ว่า context จะเป็นอะไร แต่ในภาษาธรรมชาติ ความหมายขึ้นกับ context: "bank" ใน "river bank" กับ "savings bank" ควรได้ representation ต่างกัน

Self-attention แก้ปัญหานี้โดยให้แต่ละ token "ดู" tokens อื่นทั้งหมด (หรือที่อนุญาต) แล้วสร้าง context-dependent representation — แต่ละ token มี context ใหม่ทุก layer

2. Q, K, V คืออะไร?

อุปมาจาก information retrieval: สมมติคุณค้นหาข้อมูลในฐานข้อมูล

  • Q (Query) = คำถามที่ token ปัจจุบันต้องการรู้ "ฉันต้องการข้อมูลอะไร?"
  • K (Key) = "ป้ายชื่อ" ของแต่ละ token ในฐานข้อมูล
  • V (Value) = ข้อมูลจริงที่จะดึงมา ถ้าคีย์ตรง

ทั้ง Q, K, V ล้วนมาจาก input เดียวกัน (self-attention) แต่ผ่าน linear projection ต่างกัน:

\[ Q = XW_Q,\quad K = XW_K,\quad V = XW_V \]
X ∈ ℝ^(T×d_model), W_Q, W_K, W_V ∈ ℝ^(d_model×d_k)

สำหรับโมเดลนี้: d_model=256, d_k = d_model/nhead = 256/8 = 32

3. Scaled Dot-Product Attention

สูตรหลัก
\[ \text{Attention}(Q,K,V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)V \]

ทำไมต้องหารด้วย √d_k?

สมมติ Q, K เป็น random vectors ∈ ℝ^32 ที่มี mean=0, variance=1 แต่ละ component ของ Q·K = Σᵢ QᵢKᵢ มี variance=1 รวมกัน d_k=32 items → Var(Q·K) = d_k = 32

Variance argument Var(Q·K) = Σᵢ Var(Qᵢ·Kᵢ) = Σᵢ 1 = d_k (independent components) หาร √d_k: Var(Q·K / √d_k) = Var(Q·K) / d_k = 1 → dot product มี variance=1 → softmax ไม่ saturate ที่ 0 หรือ 1
ตัวอย่าง
d_k = 32 → scale factor = 1/√32 ≈ 0.177
raw dot product ≈ 4.8 (ใหญ่ → softmax saturate) → scaled ≈ 0.85 (สมเหตุสมผล)
Q·Kᵀ Matrix Builder — 4 steps

ใช้ข้อมูลจริงจาก RL model, layer 0, head 0 สำหรับ 4 tokens แรกของ "tesbt"

4. Causal Mask

ในการ generate แบบ autoregressive token ที่ตำแหน่ง t ต้องไม่เห็น tokens ที่ตำแหน่ง t+1, t+2, ... (ยังไม่ได้ generate ขณะนั้น) — ต้องใส่ causal mask

Causal mask mechanism
\[ \text{scores}_{ij} = \begin{cases} Q_i \cdot K_j / \sqrt{d_k} & \text{ถ้า } j \le i \\ -\infty & \text{ถ้า } j > i \end{cases} \] หลัง softmax: \( e^{-\infty} = 0 \) → masked positions ให้ attention weight = 0

ใน PyTorch ใช้ torch.triu(..., diagonal=1).bool() สร้าง upper triangular mask (True = masked) แล้วส่งไปยัง nn.TransformerEncoder ผ่าน mask=

N=4: masked pairs = C(4,2) = 6 (upper triangle ไม่รวม diagonal)

5. Multi-Head Attention

แทนที่จะทำ attention ครั้งเดียวบน d_model=256 dimensions เราแยก 8 "heads" แต่ละ head มี d_k = 256/8 = 32 dimensions — ให้โมเดลมองคนละ "side" ของ representation

\[ \text{MultiHead}(X) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h) \cdot W_O \] \[ \text{head}_i = \text{Attention}(XW_Q^i, XW_K^i, XW_V^i) \]
W_O ∈ ℝ^(d_model × d_model) — merge all heads back
Multi-head split: d_model=256 → 8 heads × 32 dims

หลัง concat(8 heads) = 256 dims → ×W_O → กลับเป็น 256

6. Positional Encoding

Self-attention ไม่มีแนวคิดของ "ตำแหน่ง" — ถ้า shuffle tokens ผล attention เหมือนเดิม ต้องเพิ่ม positional information ด้วย sinusoidal encoding:

Sinusoidal PE
\[ PE(pos, 2i) = \sin\!\left(\frac{pos}{10000^{2i/d}}\right) \] \[ PE(pos, 2i+1) = \cos\!\left(\frac{pos}{10000^{2i/d}}\right) \]

ทำไม sin/cos? เพราะ PE(pos+k) เป็น linear transformation ของ PE(pos):

Proof — relative position encodable ให้ ω = 1/10000^(2i/d), PE(pos) = [sin(ωpos), cos(ωpos)] PE(pos+k) = [sin(ω(pos+k)), cos(ω(pos+k))] = [sin(ωpos)cos(ωk) + cos(ωpos)sin(ωk), cos(ωpos)cos(ωk) − sin(ωpos)sin(ωk)] เขียนเป็น matrix: PE(pos+k) = R(ωk) · PE(pos) โดย R = rotation matrix → model สามารถเรียนรู้ relative position ผ่าน linear layer บน PE
ตัวอย่าง: PE(0) vs PE(1) — dimension 0-1
ω = 1/10000^0 = 1.0
PE(0, 0) = sin(0) = 0.000
PE(0, 1) = cos(0) = 1.000
PE(1, 0) = sin(1) ≈ 0.841
PE(1, 1) = cos(1) ≈ 0.540

7. Code — PositionalEncoding และ MiniGPT.forward

model.py:9–25 — PositionalEncoding

 9class PositionalEncoding(nn.Module):
10    def __init__(self, d_model, max_len=256):
11        super().__init__()
12        pe = torch.zeros(max_len, d_model)
13        position = torch.arange(0, max_len).unsqueeze(1).float()
14
15        div_term = torch.exp(
16            torch.arange(0, d_model, 2).float() * (-math.log(10000.0) / d_model)
17        )
18
19        pe[:, 0::2] = torch.sin(position * div_term)
20        pe[:, 1::2] = torch.cos(position * div_term)
21        pe = pe.unsqueeze(0)
22        self.register_buffer("pe", pe)
23
24    def forward(self, x):
25        return x + self.pe[:, : x.size(1)]
ทำไม
line 16: div_term คำนวณ 1/10000^(2i/d) ใน log-space (torch.exp(arange × (−log10000/d))) เพื่อ numerical stability — ป้องกัน overflow สำหรับ d_model ใหญ่

line 22: register_buffer("pe", ...) ทำให้ pe ไม่ใช่ learnable parameter (ไม่อยู่ใน model.parameters()) แต่ยังคง move to device พร้อม model.to(device)

model.py:53–66 — MiniGPT.forward

53    def forward(self, x, pad_mask=None):
54        _, T = x.shape
55        causal_mask = torch.triu(torch.ones(T, T, device=x.device), diagonal=1).bool()
56
57        h = self.token_emb(x)
58        h = self.pos_emb(h)
59
60        h = self.transformer(
61            h,
62            mask=causal_mask,
63            src_key_padding_mask=pad_mask,
64        )
65
66        return self.lm_head(h)
ทำไม
line 55: torch.triu(..., diagonal=1).bool() สร้าง upper triangular mask (True = blocked) — diagonal=1 ทำให้ diagonal หลัก (i=j) ไม่ถูก mask (token เห็นตัวเอง)

mask=causal_mask ส่งเป็น mask ร่วมกัน ทุก sample ใน batch (causal constraint เหมือนกันทุก sequence), แต่ src_key_padding_mask=pad_mask ส่งแยก per-sample เพราะแต่ละ sample มี PAD positions ต่างกัน

8. Stop & Check

ทดสอบความเข้าใจ

Q1: Sequence ยาว 8 tokens มีกี่ pair (i,j) ที่ถูก mask?

เฉลย
Upper triangle ไม่รวม diagonal = n(n−1)/2 = 8×7/2 = 28 pairs
หรือ C(8,2) = 28

Q2: ถ้า d_k = 64 scale factor คือเท่าไร?

เฉลย
1/√64 = 1/8 = 0.125
เทียบกับ d_k=32: 1/√32 ≈ 0.177

Q3: ทำไม PE ถึงใช้ sin สำหรับ even dims และ cos สำหรับ odd dims?

เฉลย
sin/cos ต่าง phase 90° — แต่ละ pair (sin, cos) encode direction ในวงกลม 2D
ผลคือ PE(pos+k) = R(k)·PE(pos) โดย R เป็น rotation matrix
→ model สามารถเรียนรู้ relative position ผ่าน linear layer
